# Workflow: Brownfield UI Development

**Type:** Workflow
**Status:** Active
**Version:** 1.0.0
**When to Use:** For developing frontend/UI features in an existing project

---

## Overview

Orquestra desenvolvimento de componentes, páginas e features UI em um projeto existente.

## Fases

### 1️⃣ Planejamento

- [ ] Ler story e acceptance criteria
- [ ] Revisar designs (se existirem)
- [ ] Identificar componentes necessários
- [ ] Criar branch: `git checkout -b ui/story-{id}`

### 2️⃣ Implementação

- [ ] Criar componentes reutilizáveis
- [ ] Implementar layouts
- [ ] Integrar com API
- [ ] Implementar state management
- [ ] Adicionar validações
- [ ] Implementar error handling

### 3️⃣ Testes

- [ ] Escrever testes de componentes
- [ ] Testar user interactions
- [ ] Validar tipos
- [ ] Rodar linter
- [ ] Testar responsividade

### 4️⃣ Documentação

- [ ] Adicionar comentários em código complexo
- [ ] Documentar props de componentes
- [ ] Atualizar style guide se necessário
- [ ] Adicionar screenshots

### 5️⃣ Commit & PR

- [ ] Commit com conventional commits
- [ ] Push e criar PR
- [ ] Incluir screenshots de UI

---

**Próximos:** Review → Merge → Deploy
